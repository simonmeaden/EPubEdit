Detailed Description

A spin box that will display the value in decimal, hexadecimal, octal or binary.

Since
5.7.0
5.7.0
License: The MIT License
Copyright
© 2019 - 2020 Simon Meaden. All rights reserved.

An extended QSpinBox that can display in various common formats, (hexadecimal, octal and binary). ExSpinBox::Decimal in fact just uses the underlying QSpinBox so acts in exactly the same way.

Character case (in the case of hexadecimal) is enforced, uppercase by default, and can be changed using the setCharCase() method to either ExSpinBox::Uppercase or ExSpinBox::Lowercase.

The display type can be set using the ExSpinBox::setDisplayType() method to ExSpinBox::Decimal, ExSpinBox::Hexadecimal, ExSpinBox::Binary or ExSpinBox::Octal.

By default leading zeroes are not shown. This can be modified setting the ExSpinBox::setLeadingZeroes(bool) to true. The actual number of displayed characters is set when the minimum and maximum range is set.

◆ negBeforePrefix

ExSpinBox::negBeforePrefix
readwrite

This property holds the display negative sign before prefix flag.

Default is true. This only applies to a user supplied prefix, not the internal "0x" (Hexadecimal), "0" (Octal), "0b" (Binary) prefixes which are always supplied.

If false, the default, and assuming you set a prefix of "£" for a hexadecimal spin box then the result would be "£-0xFF" for an int value of

  1. If set to true then "-£0xFF" would result which would make more sense in this case. Of course this is a very wierd example and I cannot see any reason to display hexadecimal currency but hey ho!.
